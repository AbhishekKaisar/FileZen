Functionalies of FileZen :


* Workspace/team model: personal workspace, shared workspace, roles, invitations, access control.
* Real file system layer: upload/download, move/copy/rename/delete/restore, folder tree, favorites, recents, trash.
* Storage integration: Supabase Storage buckets plus DB metadata records for every file/version.
* Metadata pipeline: MIME detection, size, checksum, thumbnails, EXIF/media/doc metadata, extracted text, indexing timestamps.
* Search and filtering: filename search, full-text search, tags, type/date/size filters, saved views.
* Versioning and conflict handling: file versions, duplicate detection, rename/version/replace strategies.
* Organizer automation: rules based on extension, path, age, inactivity, tags, metadata, scheduled scans.
* Reports: saved report definitions, report runs, generated outputs, history.
* Settings: user settings, workspace settings, automation settings, security policies.
* Collaboration: file/folder sharing, public links, internal access, audit logs.
* Ops and trust: activity log, background jobs, scan status, error tracking, soft delete, restore flows, quotas.

Requirements of my faculty : 

Block-Based File Organization

The system should organize files into blocks.

Each block will contain specific files categorized systematically.

Day-Wise File Sorting

Files within each block must be sorted according to days of the week (e.g., Friday, Thursday, Saturday).

The sorting mechanism should ensure easy access and chronological organization.

Database with Reporting System

A database must be implemented to store and manage all file-related data.

The system should include a reporting feature that can generate structured reports based on stored data.

Android Application Interface

A user-friendly Android application interface should be developed.

The interface must allow users to interact with the system efficiently, including viewing, managing, and organizing files.















Project Proposal ( our)

2. Custom File Explorer with File System Visualization

Objectives:

To build a custom file manager that allows users to navigate directory hierarchies, visualize metadata, and perform fundamental file operations.

Key Features & OS Concepts:

Directory Navigation & CRUD: Browse root/external storage, and Create, Read, Update, and Delete files or folders.

Metadata Display: Show detailed file properties including size, block size, owner ID, and standard rwx permissions.

Search: Implement an indexing-based algorithm for efficient filename searching.

OS Concepts Applied: File systems (EXT4/F2F5), file permissions, directory structures, and storage management.

3. Automatic File Organizer Based on File Type and Metadata

Objectives:

To automate file categorization and directory management by utilizing file metadata and OS-level file handling concepts, significantly reducing manual organization effort.

Key Features & OS Concepts:

Automated Sorting: Periodically scan target directories and move files into categorized folders based on their extensions.

Metadata Extraction: Read file metadata to sort items by creation date, modification date, or specific tags.

Conflict Resolution: Automatically handle file naming conflicts during sorting (e.g appending timestamps to duplicate filenames).

OS Concepts Applied: File I/O operations, directory traversal algorithms, process scheduling, and INode management.
